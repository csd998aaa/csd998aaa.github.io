git 简介

git 是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。

分布式相比于集中式的最大区别在于开发者可以提交到本地，每个开发者通过克隆 git clone 在本地机器上拷贝一个完整的Git仓库。

以下总结在项目中使用 git 的基本操作。

分支的操作

多个并行作业时，我们会用到分支管理。在多个分支中，往往同时存在多个最新代码状态。不同分支可以进行合并，通过灵活运用多分支可以让更多人同时高效并行地进行开发。

如下图 master dev testing 三个分支：

一般来说：

master 默认分支，用于生产环境部署。

dev 开发分支，用于日常开发。

testing 测试分支，用于测试环境及bug等测试。

一个仓库当中，master 为默认分支，它担任稳定分支的角色，如用于生产环境。其他分支与它合并之后，就可以保证 master 分支为最新内容。
