块格式上下文（BFC）是怎么回事？

Posted on 2020-03-07

MDN解释：它是Web页面的可视化CSS渲染的一部分，是块盒子的布局过程发生的区域，也是浮动元素与其他元素交互的区域(如:非BFC元素重叠)。

首先，格式化上下文有以下分类：
① BFC (Block Formatting Contexts)译为”块级格式化上下文”
② IFC (Inline Formatting Contexts)译为”内联格式化上下文”
③ GFC (GridLayout Formatting Contexts)译为”网格布局格式化上下文”
④ FFC (Flex Formatting Contexts)译为”自适应格式化上下文”

本文重点介绍「BFC」

BFC的形成条件：

① float 的值不为 none
② position 的值不为 static 或者 relative
③ display 的值为 table, table-cell, table-caption, inline-block, flex, inline-flex
④ overflow 的值不为 visible

那么，BFC会产生什么影响，或者它有什么作用？

同一BFC会导致的外边距折叠

注：产生折叠的必备条件：margin必须是邻接的!

现在我们来对会产生外边距折叠的情况进行分析~

要产生外边距折叠，需满足以下条件：
① 处于常规文档流，处于同一个 BFC 中
② 没有 padding 和 border 将他们分隔开

我们可以使用BFC来包含浮动

初学者经常碰到使用 float 后导致高度塌陷的问题。容器元素没有高度，它的浮动孩子将会脱离页面的常规流，可以通过定义一个BFC来达到清除浮动。

举个栗子：

现在的box1有高度，因为被box2撑开了，因为他们都处在普通文档流。但是只要我们给box2加了float，情况就不一样了。

外层的box1高度塌陷了。那么我们的解决方法就是触发BFC。

给box1加了overflow: hidden，就解决了高度塌陷问题。

使用BFC来防止文字环绕

这是我们经常看到的利用图片浮动来制造文字环绕效果，但是有的时候我们并想出现这种效果，因为这样文字像膏药一样和图片贴在一起，不想他环绕图片。我们可以触发BFC，即给p标签加一个 overflow：hidden 。

在多列布局中使用BFC

一个多列布局占满了整个容器的宽度，最后一列有时候将会被挤到下一行。然而，如果我们在一个列的布局中建立了一个新的BFC，它将会在前一列填充完之后的后面占据所剩余的空间。
